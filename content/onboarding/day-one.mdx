# Day-1 Checklist

Work through this top to bottom. Every access item is granted by the **lead &
technical owner** ([who to ask](/help/who-to-ask)) — request them all in one
message.

## Accounts & access

- [ ] **GitHub** — invitation to the `adawy-group` organization
- [ ] **Trello** — the department board (read [The Board & Labels](/how-we-work/the-board) first)
- [ ] **Figma** — the Adawy design files and design system
- [ ] **Vercel** — team access (preview/staging/production visibility)
- [ ] **Sentry & Better Stack** — error tracking and uptime dashboards
- [ ] **Notion** — internal documentation workspace
- [ ] **Team password manager** — this is the only place credentials live;
      [never in chat or git](/standards/security)
- [ ] Company email / communication channels

## Machine setup

- [ ] **Node.js** (current LTS) via a version manager — `fnm` or `nvm`
- [ ] **Git**, configured with your name and work email
- [ ] An editor with TypeScript, ESLint, and Prettier integration
- [ ] Clone the platform repo and this docs repo (see
      [Dev Environment Setup](/onboarding/environment))

## Context (read in this order)

- [ ] [About Adawy Group](/about) — what the business is and why software exists in it
- [ ] [How We Work](/how-we-work) — the workflow you'll live in from today
- [ ] [Products & Architecture](/architecture) — what we're building and why

By the end of day one you should be able to run a project locally and explain
to a friend what Adawy Group does and what the department builds for it.
