# Your First PR

Target: **merged within your first week.** The full loop matters more than the
size of the change.

## 1. Pick the card

Your first card comes from the top of **This Week** — the lead flags starter
cards for new joiners. Good first PRs: a stub page in this handbook, a small
Content card, a Maintenance/grey card. The card must be in **Ready for Dev**
(design approved + fully specified) before you start —
[that's the rule](/how-we-work/definition-of-done) even for small work.

## 2. Branch and build

```bash
git checkout main && git pull
git checkout -b fix/navbar-rtl-alignment   # type prefix matches the card's label
```

Small commits, written for the reader. Check the
[coding conventions](/standards/coding) as you go — strict TS, design tokens,
accessibility, no raw `<img>`.

## 3. Open the PR

- Title says what it does; description says **why**, and links the Trello card.
- Paste the PR link back onto the card.
- Wait for the automated checks and the preview deploy — then click through
  your own preview before requesting review. Reviewers review working pages,
  not promises.

## 4. Review → merge → verify

The lead reviews within one working day. Address feedback with new commits
(don't force-push during review). After merge: confirm the change on staging,
check Sentry is quiet, move the card to **Done**.

## 5. You shipped

You've now touched every part of the system you'll use daily: the board, the
repo, the checks, the preview pipeline, review, staging, and Done. Everything
after this is the same loop with bigger cards.

> Found anything confusing during this process? The docs failed you — fixing
> them is your natural second PR.
