# Standards

Conventions that keep our codebases consistent so anyone can work in any repo:

- **[Coding Conventions](/standards/coding)** — TypeScript style, naming, patterns.
- **[Project Structure](/standards/project-structure)** — how repos are organized.
- **[Security Basics](/standards/security)** — the non-negotiables.
- **[UI & Components](/standards/ui)** — design system and component rules.

Every standard here states **why it exists**. A rule without a reason is a
candidate for deletion.
