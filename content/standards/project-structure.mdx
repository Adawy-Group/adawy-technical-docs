# Project Structure

## The platform layout

The platform is one repository. The structure reflects the multi-tenant
architecture — shared first, brand-specific as thin configuration:

```
platform/
├─ app/                  # Next.js App Router — routes shared by all brands
├─ components/           # The shared design system (the product, not a utility)
│  ├─ ui/                # Primitives (shadcn/ui based)
│  └─ sections/          # Composed page sections: heroes, galleries, cards…
├─ templates/            # Complete page templates: industrial / creative / services / retail
├─ brands/               # One folder per brand: tokens, fonts, logo, config — no logic
├─ lib/                  # Shared utilities, Odoo client, helpers
└─ content/              # Site content (MDX/config) — edited via PR
```

## The rules behind the layout

1. **Brand folders contain configuration, not code.** Colors, typography,
   logos, navigation structure. If you are writing a component inside
   `brands/`, stop — it belongs in the design system, themed by tokens.
   *Why:* this is what makes brand #39 a configuration task
   ([ADR-001](/architecture/adrs)).
2. **Templates are the unit of reuse.** New sub-brand sites pick a template
   and configure it. Custom layouts are for flagships only (corporate site,
   Adawy Studio) and are explicitly scoped. *Why:* one "small" one-off
   customization replicated across 38 brands is a maintenance disaster.
3. **The design system is documented as it grows.** A component without usage
   guidance is half-finished — the next 35 sites are built by people who
   weren't there when it was written.
4. **Monorepo by default.** The brand sites, design system, and CMS live
   together so a change can be reviewed and shipped atomically. The backend
   service (NestJS) is its own repo because it deploys to different
   infrastructure on a different cadence.

## This documentation site

```
adawy-technical-docs/
├─ app/                  # Nextra app shell ([[...mdxPath]] catch-all + layout)
├─ content/              # All handbook pages (MDX) — sidebar mirrors folders
│  └─ <section>/_meta.ts # Sidebar order & titles per section
├─ next.config.mjs       # Nextra config
└─ mdx-components.tsx
```

To add a page: create the `.mdx` file in the right section folder, add it to
that folder's `_meta.ts`, open a PR.
